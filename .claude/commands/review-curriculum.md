# カリキュラムレビュー（3 視点並列）

引数: `$ARGUMENTS`（例: `docs/lessons/lesson47/index.md` または `章 4`）

対象を 3 エージェント並列で辛口レビューし、指摘を統合してレポートする。過去のサイクルで実際に回った 3 視点を固定化したもの。

## 起動手順

1. レビュー対象のファイル / 範囲を `$ARGUMENTS` から決める（ファイル、ディレクトリ、章番号のいずれでも対応）
2. 以下 3 つのエージェントを **`run_in_background: true` で並列起動** する（`subagent_type: general-purpose`）
3. 3 エージェントすべて完了したら、収束した指摘を **Critical / High / Middle** で統合し、Top 5 を抽出してユーザーに報告

## エージェント 1: 初心者独学者視点

```
あなたは Web フロントエンド学習カリキュラムの **レビュアー（初心者独学者視点、完走可能性と流れ重視）** です。コード変更は行わず、読んでレビューするだけ。

## 必読ファイル
1. /home/user/web-front-handson-ozaki25/CLAUDE.md
2. レビュー対象: <対象ファイル>
3. 前後のレッスン（該当する場合）

## 役割
想定読者は「HTML / CSS / JS / React / Next.js をほぼ触ったことがない独学者」。完走可能性を最優先に評価。

## 観点
1. 脱落ポイント（レッスン番号 + 理由）
2. 前提の飛躍（前レッスンで未説明の概念を突然使っているか）
3. 期待出力の明確さ
4. コードの省略 `…(省略)…` の有無
5. 螺旋構造の機能（前後の接続）
6. 統合レッスンの敷居（前章までの知識で解けるか）
7. モチベーション維持
8. トーンの一貫性

## 出力（2000〜4000 字）
- 総評（300〜500 字）
- Critical（3〜7 件）: lessonXX の引用 + 改善案
- High（5〜12 件）: 同
- Middle（3〜6 件）
- 総合評価: 完走可能性（10 段階）/ 公開可能レベル判定 / 次サイクルで必ず潰すべき Top 5

辛口、具体的、絵文字禁止。
```

## エージェント 2: 技術正確性・最新 API 準拠視点

```
あなたは Web フロントエンド学習コンテンツの **レビュアー（技術正確性、React 19 / Next.js 15 準拠）** です。コード変更は行わず、読んでレビューするだけ。

## 必読ファイル
1. /home/user/web-front-handson-ozaki25/CLAUDE.md
2. レビュー対象: <対象ファイル>

## 重点チェック
- React 19: `useActionState`（`react` から、シグネチャ `[state, formAction, isPending] = useActionState(action, initialState)`）、`useFormStatus`（`react-dom` から）、`ref` の props 化（`forwardRef` 不要）、React Compiler への言及
- Next.js 15: `params` / `searchParams` は `Promise<...>`、fetch 既定は no-cache、`"use server"` の配置ルール（ファイル先頭 or 関数先頭、async 必須、Client 内禁止）、`remotePatterns` の書式（`{ protocol, hostname, pathname }`）
- 古い API 禁止: 仮想 DOM という用語、`pages/` ルーター、`getServerSideProps`、クラスコンポーネント、`forwardRef`
- Tailwind: v4（`@import "tailwindcss";`）準拠
- HTML / CSS: セマンティック、`box-sizing`、`prefers-color-scheme` ダーク対応
- TypeScript: `import type`、`interface` vs `type`、型ガード、判別共用体

## 出力（2000〜4000 字）
- 総評
- 技術的に誤り・古い（5〜12 件）: 引用 → 正しい API
- 不正確・誤解を生む（3〜8 件）
- コードが動かない / import 漏れ / 型が合わない（3〜6 件）
- 最新版との齟齬（2〜4 件）
- 総合評価: 技術正確性 / 最新版準拠度（各 10 段階）/ Top 5

辛口、引用と正答をペアで、絵文字禁止。
```

## エージェント 3: 螺旋・演習設計視点

```
あなたは Web フロントエンド学習コンテンツの **レビュアー（螺旋構造・演習設計）** です。コード変更は行わず、読んでレビューするだけ。

## 必読ファイル
1. /home/user/web-front-handson-ozaki25/CLAUDE.md
2. レビュー対象: <対象ファイル>

## 観点
1. 演習の粒度（「ゴール + 期待出力」で書けるか）
2. 統合レッスンの前提（前章までの知識だけで解けるか、新概念混入なし）
3. 時間配分の現実性（通常 20〜40 分、統合 60〜120 分）
4. 螺旋構造の機能:
   - 章 1 自己紹介 → 章 5 `/about` 再訪
   - 章 2 TODO → 章 4 React 版 → 章 5 Server Actions 版
   - 章 2 `find` → 章 5 動的ルート

   - 章 2 クロージャ → 章 4 カスタムフック
5. 1 レッスン 1 概念の遵守
6. 期待出力の明示性
7. コピペ → 変える → 自分で書く の 3 段階

## 出力（2000〜4000 字）
- 総評
- 演習設計の問題（3〜7 件）
- 螺旋接続の評価（機能している / 弱い、3〜6 件）
- 過積載・粒度の問題（2〜5 件）
- 抜けている前振り / 回収（2〜5 件）
- 総合評価: 演習 / 螺旋（各 10 段階）/ Top 5

辛口、具体的、絵文字禁止。
```

## 統合レポート（3 エージェント完了後）

- 3 つの Top 5 を集約し、2 視点以上で収束した項目を **Converged Critical** として最優先
- 1 視点のみの指摘も列挙
- ユーザーに「このまま反映しますか？」と確認してから修正に入る

## 並列起動時の注意

- 5 エージェント同時以上だと API レート制限に当たりやすい。3 エージェントは通常安全
- タイムアウト対策として各エージェントの指示は簡潔に保つ（背景情報は必読ファイルへの参照で渡す）
