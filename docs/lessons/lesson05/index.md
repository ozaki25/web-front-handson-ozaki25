# lesson05: ページの骨格を組む

## ゴール

- `<header>` / `<main>` / `<footer>` でページを大きな 3 つのブロックに分けられる。
- 見出し階層の原則（`<h1>` はページに 1 つ、段を飛ばさない）を意識してページを書ける。
- アクセシビリティの最低限（意味のあるタグを使う、コントラスト、フォーカスリングを消さない）について 1 段落で説明できる。

## 解説

### いままでは `<body>` に全部並べていた

前のレッスンまで、自己紹介ページの内容はすべて `<body>` の直下に並んでいました。ページの要素が少ないうちはこれでも動きますが、情報が増えると「どこからどこまでがヘッダーで、どこが本文か」が分かりにくくなります。

HTML には、ページの「意味の塊」を表す **セマンティックタグ** が用意されています。画面上の見た目は `<div>`（意味のない箱）と変わりませんが、「これはヘッダー」「これは本文」「これはフッター」といった情報を持っているので、ブラウザやスクリーンリーダー、検索エンジンが構造を理解しやすくなります。

### 代表的な骨格タグ

- `<header>`: ページやセクションの冒頭部分。サイトロゴ、ナビゲーション、タイトルなど。
- `<main>`: ページの **主要コンテンツ**。原則としてページに 1 つ。
- `<footer>`: ページやセクションの末尾部分。著作権表記、関連リンクなど。
- `<nav>`: ナビゲーションリンクの集まり。メインメニューなど。
- `<section>`: 章・節。見出しを伴う意味の塊。

基本の形はこうなります。

```html
<body>
  <header>
    <h1>サイト名 / ページのタイトル</h1>
    <nav>
      <a href="/">ホーム</a>
      <a href="/about">自己紹介</a>
    </nav>
  </header>
  <main>
    <p>ページの主要コンテンツをここに書きます。</p>
  </main>
  <footer>
    <p>&copy; 2026 私の名前</p>
  </footer>
</body>
```

### 見出し階層のおさらい

「テキストを書く」で見出しは「意味の階層」だと書きました。骨格タグと合わせるときも、同じルールで考えます。

- ページの主題を表す `<h1>` は **1 つだけ**。このコースでは `<header>` の中に置く。
- `<main>` の中の各セクション（自己紹介の本文、好きなもの、やりたいこと、など）は `<h2>` で始める。さらに下の階層は `<h3>`。
- 段を飛ばして書かない（`<h1>` → `<h3>` のようにしない）。

DevTools の「Accessibility」パネル（Elements パネル内のタブ）を開くと、ページの見出しが一覧で見えます。見出しだけ見てページの構造が伝わるかが、良い目安になります。

### `<div>` と `<section>` の違い

ここで 1 つだけ補足します。「ボックスモデルで余白を作る」で `<div>` も出てきますが、違いを一言で言うと次の通りです。

- `<div>`: 意味を持たないただの箱。CSS でまとめて装飾したいときに使う。
- `<section>`: 見出しを伴う意味の塊。章・節に相当する。

迷ったら、「この塊には見出しが付くか？」で判断します。付くなら `<section>`、付かない純粋な装飾の入れ物なら `<div>`。

## アクセシビリティ最小限（コラム）

### 意味のあるタグを使う

`<div>` や `<span>` でどれだけ見た目を整えても、スクリーンリーダーはその塊が「ヘッダー」なのか「本文」なのかを判断できません。`<header>` / `<main>` / `<footer>` / `<nav>` のような意味のあるタグを使うと、読み上げの飛ばし読み（「ヘッダーまでスキップ」など）が使えるようになります。見出しも同じ理由で、`<h2>` と分かって使う方が見栄えだけの「大きな文字」よりずっと価値があります。

### コントラスト（配色）

文字と背景の明暗差が弱いと、視力が弱い人にとって文字が読めなくなります。例えば薄いグレーの文字を白背景に置くと、年配の方やまぶしい屋外では読み取れません。このコースで CSS の色を指定するとき（「色と文字を整える」以降）は、**白背景だけを前提にしない** ようにします。ダークモードでも見える色を選ぶのが基本です。

### フォーカスリングを消さない

キーボードで Tab キーを押しながらページを移動すると、ボタンやリンクの周囲に青い枠（フォーカスリング）が表示されます。これは「いまここにフォーカスがある」を示す大事なヒントです。CSS で `outline: none` と書くとこの枠を消せますが、**消してしまうとキーボード操作ができなくなる** 人が出ます。「デザイン上どうしても」というときは、別の見た目（太い下線や背景色の変化）で代わりの合図を付けます。「クラスと状態」で `:focus` を扱うときに、この原則を改めて守ります。

### 前レッスンの状態から始める

前のレッスンで作ったプロジェクトを開き直します。`index.html` は、プロフィール画像・好きなもの・やりたいこと・お気に入りサイトが入った状態です。

### コピペで動かす

骨格タグで全体を整理します。`<body>` の中身を次の構造に置き換えます。

```html
<!DOCTYPE html>
<html lang="ja">
  <head>
    <meta charset="UTF-8" />
    <title>自己紹介</title>
  </head>
  <body>
    <header>
      <h1>オザキの自己紹介</h1>
      <nav>
        <ul>
          <li><a href="#profile">プロフィール</a></li>
          <li><a href="#likes">好きなもの</a></li>
          <li><a href="#goals">今年やりたいこと</a></li>
          <li><a href="#links">お気に入りサイト</a></li>
        </ul>
      </nav>
    </header>

    <main>
      <section id="profile">
        <h2>プロフィール</h2>
        <img
          src="https://placehold.jp/200x200.png"
          alt="オザキのプロフィール画像(仮)"
          width="200"
          height="200"
        />
        <p>はじめまして。Web フロントエンドを勉強中です。</p>
        <p>
          いまは <strong>HTML の基礎</strong> を学んでいます。読むだけでなく、<em>自分でも手を動かして</em> 覚えていきたいです。
        </p>
      </section>

      <section id="likes">
        <h2>好きなもの</h2>
        <ul>
          <li>コーヒー</li>
          <li>散歩</li>
          <li>本</li>
        </ul>
      </section>

      <section id="goals">
        <h2>今年やりたいこと</h2>
        <ol>
          <li>Next.js で小さなアプリを作る</li>
          <li>毎週 1 本ブログを書く</li>
          <li>早起きする</li>
        </ol>
      </section>

      <section id="links">
        <h2>お気に入りサイト</h2>
        <ul>
          <li>
            <a href="https://developer.mozilla.org/ja/" target="_blank" rel="noopener">MDN Web Docs（日本語）</a>
          </li>
          <li>
            <a href="https://ja.react.dev/" target="_blank" rel="noopener">React 公式（日本語）</a>
          </li>
          <li>
            <a href="https://nextjs.org/" target="_blank" rel="noopener">Next.js 公式</a>
          </li>
        </ul>
      </section>
    </main>

    <footer>
      <p>&copy; 2026 オザキ</p>
    </footer>
  </body>
</html>
```

補足:

- `id` 属性は要素に名前を付ける仕組み。`href="#profile"` のようにリンク先に書くと、同じページ内の `id="profile"` の要素までスクロールする「ページ内リンク」になる。
- `&copy;` は HTML の **実体参照** と呼ばれる書き方で、画面上では `©` 記号になる。

### 期待出力

- ページの一番上に「オザキの自己紹介」とナビゲーションのリンク 4 つ。
- その下に、4 つのセクション（プロフィール / 好きなもの / 今年やりたいこと / お気に入りサイト）が縦に並ぶ。各セクションに `<h2>` の見出しが付く。
- ページの一番下に `© 2026 オザキ` と表示される。
- ナビの「好きなもの」をクリックすると、同じページ内の「好きなもの」セクションまでスクロールする。
- DevTools の Elements パネルで、`<header>` / `<main>` / `<footer>` / `<nav>` / `<section>` のタグがツリーに見える。

### 変えてみる

1. `<nav>` の中のリンク文言を、自分の好みの言い回しに変える（「私について」「好きなもの」など）。
2. 新しいセクション `<section id="contact">` を `</main>` の直前に追加し、`<h2>連絡先</h2>` と短い紹介文を書いてみる。`<nav>` にもその項目を追加する。
3. `<footer>` に自分の名前と今年の年号を入れる。

### 自分で書く

次の構造を何も見ずに `<body>` の中に書いてみます。

- `<header>`: ページタイトル `<h1>今週の記録</h1>`
- `<main>`: `<section>` を 2 つ（「月曜」と「火曜」、それぞれ `<h2>` と短い `<p>`）
- `<footer>`: 自分の名前

書いたあと、このレッスンの例と見比べてください。`<header>` / `<main>` / `<footer>` の順序、`<section>` の中に `<h2>` と本文を入れる形、が守れていれば OK です。

## まとめ

- `<header>` / `<main>` / `<footer>` はページの意味の塊を表すタグ。見た目は `<div>` と同じだが、構造が伝わる点で価値がある。
- `<h1>` はページに 1 つ。`<main>` の中の章は `<h2>` から始めて段を飛ばさない。
- アクセシビリティの最小限: 意味のあるタグを使う、コントラストに気を配る、フォーカスリングを消さない。
