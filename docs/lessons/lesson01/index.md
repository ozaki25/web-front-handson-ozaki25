# lesson01: HTML ってなに？

## ゴール

- HTML が何を表すものなのかを、自分の言葉でざっくり説明できる。
- もっとも小さな HTML ファイルを書いてブラウザで開ける。
- ブラウザの DevTools（開発者ツール）を開き、Elements パネルで書いた HTML を覗ける。

## 解説

### HTML は「ページの構造を書くための言語」

Web ページは、文字・画像・リンク・入力欄などの部品を組み合わせて出来ています。HTML（HyperText Markup Language）は、その部品ひとつひとつに「これは見出し」「ここは段落」「ここはリンク」といった意味のラベルを付けていく言語です。

ブラウザは HTML を上から順に読み、書かれているラベル（タグ）を元に画面を組み立てます。この「タグを使って意味を付けていく作業」を **マークアップ** と呼びます。

### タグの基本

HTML の部品は次のような形で書きます。

```html
<p>こんにちは</p>
```

- `<p>` を **開始タグ**、`</p>` を **終了タグ** と呼ぶ。
- 開始タグから終了タグまでをまとめて 1 つの **要素** と呼ぶ。
- `<p>` は paragraph（段落）の頭文字。

一部のタグは終了タグを書かない（例: `<img>`、`<br>`）。これは後のレッスンで出てきます。

### 最小の HTML の骨格

ブラウザに「これは HTML5 のページです」と伝えるためのおまじないが、ファイルの先頭に必要です。

```html
<!DOCTYPE html>
<html lang="ja">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1" />
    <title>ページのタイトル</title>
  </head>
  <body>
    <h1>見出し</h1>
    <p>本文</p>
  </body>
</html>
```

**`<meta name="viewport">`** はスマートフォンで等倍表示するための指定です。これがないと、モバイル端末では「PC 画面を縮めて表示」する挙動になり、文字が極小になって読めません。**初出のテンプレに必ず入れる定型** として覚えておきます。以降のレッスンでもこの骨格を土台に使います。

それぞれの意味はこうです。

- `<!DOCTYPE html>`: このファイルは HTML5 である、という宣言。先頭の 1 行に必ず書く。
- `<html lang="ja">`: ページ全体の入れ物。`lang="ja"` で「日本語のページ」と指定する。
- `<head>`: ページの設定情報を入れる場所。画面には表示されない。文字コード（`<meta charset="UTF-8">`）や、タブに出るタイトル（`<title>`）を書く。
- `<body>`: 画面に表示される本文を入れる場所。見出しや段落はここに書く。
- `<h1>`: そのページで 1 番大きな見出し。heading の頭文字。
- `<p>`: 段落。

HTML は **入れ子**（ネスト） 構造です。`<html>` の中に `<head>` と `<body>` があり、`<body>` の中に `<h1>` や `<p>` がある、というツリーになっています。インデント（字下げ）を揃えると、親子関係が読みやすくなります。

### DevTools を開く

DevTools（開発者ツール）は、ブラウザに付いている「ページの中身を覗くための道具箱」です。このコース全体で何度も開くので、開き方を覚えておきます。

- Windows / Linux: キーボードの `F12` キー、または `Ctrl + Shift + I`
- macOS: `Cmd + Option + I`
- どの OS でも: ページ上で右クリック → 「検証」または「要素を検証」

開いたらパネルの中から「Elements」（または「要素」）タブを選びます。ここには、ブラウザが読み込んだ HTML のツリーがそのまま表示されます。要素をクリックすると、画面上の対応部分がハイライトされ、どの HTML がどこに対応しているのかが見えます。

## 演習

### 途中から始める場合

このコースの最初のレッスンです。手ぶらで始めて問題ありません。

### 使う環境

このコースではコードを **StackBlitz** というサービスで動かします。ブラウザだけで起動でき、インストールは不要です。GitHub でログインすると保存や Fork（自分のコピーを作る）も使えるようになります（必須ではありません）。

### 手順

1. <https://stackblitz.com/> を開く。
2. トップページに並ぶテンプレートカードの中から **HTML / CSS / JS**（Vanilla） を選ぶ（HTML5 のロゴが目印。TypeScript と書かれたものではない方）。カードが見当たらないときは画面の検索ボックスに `html` と入れて絞り込むか、直リンク <https://stackblitz.com/edit/web-platform> を開く。
3. 最初から `index.html` が用意されている。中身を一度すべて消して、下のコードに置き換える。

```html
<!DOCTYPE html>
<html lang="ja">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1" />
    <title>はじめての HTML</title>
  </head>
  <body>
    <h1>Hello, HTML</h1>
    <p>これは最初のページです。</p>
  </body>
</html>
```

保存するとプレビューが自動で更新され、右側のプレビューに **大きな文字で「Hello, HTML」、その下に「これは最初のページです。」** と表示されます。

### 期待出力

- プレビュー画面に見出しと段落が並ぶ。
- ブラウザのタブ名（プレビューを新しいウィンドウで開いた場合）が「はじめての HTML」になる。

### DevTools で確認する

1. プレビュー右上にある「Open in New Tab」アイコンをクリックし、別タブでページを開く。
2. そのタブで `F12`（または上で書いた他のショートカット）を押して DevTools を開く。
3. 「Elements」タブを選ぶ。
4. ツリーから `<h1>Hello, HTML</h1>` を探してクリックする。プレビュー側の「Hello, HTML」がハイライトされれば成功。
5. 同じ要領で `<p>` をクリックし、段落がハイライトされることも確認する。

### 変えてみる

1. `<title>` の中身を自分の名前に変えてみる。タブのタイトル表示が変わることを確認する。
2. `<h1>` の文字を好きな言葉（例: `自己紹介`）に変えてみる。プレビューが更新されることを確認する。
3. `<p>` をもう 1 行増やしてみる。2 つの段落が並ぶことを確認する。

```html
<p>これは最初のページです。</p>
<p>今日から HTML を書きはじめます。</p>
```

### 自分で書く

いちどタブを全部閉じて、新しい StackBlitz プロジェクトをもう 1 つ作る。何も見ずに、骨格（`<!DOCTYPE html>` から `</html>` まで）を自分で打ち込んでみる。書いたあとに本文の最小 HTML と見比べて、足りないタグがあれば補う。

はじめは `<!DOCTYPE html>` や `<meta charset="UTF-8">` を忘れても構いません。「無いとどうなるか」は DevTools で Warning（警告）として出ることが多いので、それを見ながら直していけば身につきます。

## まとめ

- HTML はページの部品に意味のラベルを付ける言語で、タグで書く。
- ファイルの先頭に `<!DOCTYPE html>`、全体を `<html>` で包み、中に `<head>` と `<body>` を置くのが最小の骨格。
- DevTools の Elements パネルで、書いた HTML がブラウザにどう読み取られているかを目で確かめられる。
- 別のレッスンで、`<body>` の中身を増やして、見出し・段落・強調を使い分けて自己紹介ページの書き出しを作っていく。
