## ブランチ運用（CLAUDE.md 9 章）

- `draft`: 下書き（**現在ここで作業中**）。`docs/drafts/` のみ編集可。
- `publish/lessonXX`: main に出す時のレッスン単位ブランチ（今回はまだ使わない）。
- `main`: 公開済みのみ。**直接書き込み禁止**。

コミットは draft に積む。push は `git push -u origin draft`。

### フック注意点

- PR 関連のフック（`validate-pr.sh`）もあるが、今は関係しない。

## コンテンツ作成フェーズの方針

各レッスンは CLAUDE.md 7 章のテンプレートに厳密に従う:

```
# lessonXX: タイトル

## ゴール
## 解説
## 演習
## まとめ
```

### 必須守るべき原則（CLAUDE.md 6 章）

1. 座学 → 演習
2. **コードは省略しない**（import も閉じタグも）
3. 知らない用語を前提にしない
4. 期待出力を必ず示す
5. コピペ → 変える → 自分で書く
6. 最新版を前提

### 避けたい書き方（CLAUDE.md 11 章）

- 長い前置き
- 読者前提の省略（「ご存知の通り」等）
- 古い API（旧 `pages/` ルーター、クラスコンポーネント等）
- レベル跳ね上がり
- `…(省略)…`

### 分量の目安

1 レッスンあたり 200〜500 行の markdown（新概念の複雑さで変動）。統合レッスンは 500〜800 行。

### 置き場所

`docs/drafts/lessons/lessonXX/index.md`

（注: `docs/lessons/` は publish 時のみ使用。作業中は `docs/drafts/` 配下）

### 並行化の提案

53 レッスン全部を 1 エージェントで直列に書くと時間がかかる。章ごとに subagent（general-purpose）を割り振って並行作成する戦略を検討:

- 章 1: 1 エージェント
- 章 2: 1 エージェント
- 章 3: 1 エージェント
- 章 4: 1 エージェント
- 章 5: 1 エージェント

ただし章間の依存（螺旋構造）があるため、各エージェントに「前章のレッスンで扱った概念リスト」「共通題材の状態」「lesson-list.md のパス」を明示的に渡す必要あり。

あるいは、メインエージェントが章ごとに順に書いていく（前章の完成を踏まえて次章を書ける）方が品質は上がる可能性。

### サイドバー・config.mts の更新

コンテンツ作成後、`docs/.vitepress/config.mts` のサイドバーに lesson link を追加。ただし draft ブランチでは `docs/drafts/` にあるので、publish 時に `docs/lessons/` へ移動する必要がある点に注意。

draft ブランチでも動作確認したい場合は、サイドバー config を一時的に drafts パスに向けるか、VitePress ルートを変更する。

## コミット・push ルール

- Stop hook（`~/.claude/stop-hook-git-check.sh`）が uncommitted 変更を検出するので、作業一段落ごとにコミット・push する
- コミットメッセージは日本語短文で「〜を追加」「〜を更新」など
- 署名フッター（`https://claude.ai/code/session_xxx` 形式）は **不要**（このリポジトリでは付けていない）
- push は `git push -u origin draft`

## ユーザーの要望・好み（過去の会話から）

- 対話的に合意を取ることを重視（ただし「全部作っていい」と明示された以降は自律的に進めてよい）
- 粘り強く確認する姿勢を好む
- 余計な絵文字・装飾を嫌う
- Markdown はシンプルに、表は表で、箇条書きは箇条書きで
- テーマカラー: ダークブルー（WCAG AA 4.5:1 以上）
- サイドバー区切り線などの VitePress デフォルト挙動は尊重
- タイトル・見出しは体言止めで直接的（「Web の素材」のような詩的表現は嫌う）

## レビューエージェントのプロンプト雛形（参考）

各エージェントを `general-purpose` subagent で `run_in_background: true` で並列起動する。プロンプトには以下を必ず含める:

- 役割（どの視点でレビューするか）
- 読むファイルのパス（`docs/drafts/lesson-list.md`、`CLAUDE.md`）
- レビュー観点（5〜7 項目）
- 出力フォーマット（見出し・字数目安）
- 「辛口で、具体的に」の指示

サイクル 1 / 2 のプロンプトは会話履歴にあるのでそちらを流用・改変する。
